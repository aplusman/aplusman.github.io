VI. A TAXONOMY OF PROBLEMS
A. SCENE CLASSIFICATION
1) PROLOGUE AND SIGNIFICANCE
2) CHALLENGES
--> How to choose the 2.5/3D data representation that provides maximum information with minimum computational complexity.
--> How to distinguish between fine-grained categories and appropriately model intra-class variations.
--> How to design algorithms that can handle illuminations, background clutter and 3D deformations.
--> How to design algorithms that can learn from limited data.
3) METHODS OVERVIEW
B. OBJECT DETECTION
1) PROLOGUE AND SIGNIFICANCE
2) CHALLENGES
--> Object identification in Highly cluttered real-world environments can be very challenging.
--> Viewpoint and illuminations variations should be handled in detection algorithms.
--> In many scenarios, it is necessary to understand the scene context to successfully detect objects.
--> Objects categories have a long-tail (imbalanced) distribution, which makes it challenging to model the infrequent classes.
3) METHODS OVERVIEW

C. SEMANTIC SEGMENTATION
1) PROLOGUE AND SIGNIFICANCE
--> labelling
2) CHALLENGES
--> Object identification in Highly cluttered real-world environments can be very challenging.
--> Viewpoint and illuminations variations should be handled in detection algorithms.
--> In many scenarios, it is necessary to understand the scene context to successfully detect objects.
--> Objects categories have a long-tail (imbalanced) distribution, which makes it challenging to model the infrequent classes.
3) METHODS OVERVIEW

D. PHYSICS-BASED REASONING
1) PROLOGUE AND SIGNIFICANCE
--> infer hidden dynamics in a scene.
2) CHALLENGES
--> How to start with very limited information (e.g., a still image) and how to perform extrapolation to predict rich information about scene dynamics.
--> How to adequately model prior information about the physical world.
--> How to reason about the contextual information.
3) METHODS OVERVIEW
a: DYNAMICS PREDICTION
b: SUPPORT RELATIONSHIPS
c: STABILITY ANALYSIS
d: HAZARD DETECTION
e: OCCLUSION REASONING

E. OBJECT POSE ESTIMATION
1) PROLOGUE AND SIGNIFICANCE
2) CHALLENGES
--> Simultaneous object detection and orientation estimation is challenging.
--> Object's pose can vary significantly from one scene to another, therefore the algorithm should be invariant to these changes.
--> Occlusions and deformations make the pose estimation task difficult especially when multiple objects are simultaneously present.
3) METHODS OVERVIEW

F. 3D RECONSTRUCTION FROM RGB-D
1) PROLOGUE AND SIGNIFICANCE
2) CHALLENGES
--> Complete 3D reconstruction from incomplete information is an ill-posed problem with no unique solution.
--> sensor noise, low depth resolution, missing data, and quantization errors.
--> It requires external information incorporation about the scene or object geometry for a successful reconstruction.
3) METHODS OVERVIEW

G. SALIENCY PREDICTION
1) PROLOGUE AND SIGNIFICANCE
--> to detect some salient regions corresponding to important objects and events in a scene and mutual relationships.
2) CHALLENGES
--> Because saliency is a complex function of different factors including appearance, texture, background properties, location, depth etc., it is a challenge to model these intricate relationships.
--> How to accurately model object saliency from both top-down and bottom-up cues.
--> How to adequately encode the local and global context.
3) METHODS OVERVIEW

H. AFFORDANCE PREDICTION
1) PROLOGUE AND SIGNIFICANCE
--> Affordance defines the possibility of an agent to perform actions with an object.
2) CHALLENGES
--> How to acquire information (e.g., multiple sources and reason) to discover relationships.
--> How to model the hidden context (e.g., humans not present in the scene) to predict the correct affordances of objects.
--> How to reason about physical and material properties.
3) METHODS OVERVIEW

I. HOLISTIC/HYBRID APPROACHES
1) PROLOGUE AND SIGNIFICANCE
2) CHALLENGES
--> How to model the relationships between objects and background if inter-object interactions exist.
--> How to execute efficient training and inference from reasoning at multiple levels of scene decomposition.
--> How to integrate multiple individual tasks.
3) METHODS OVERVIEW
